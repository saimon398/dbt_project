#### Описание модели
Таблица-линк "Курьер — Заказ". Хранит уникальные связи между курьерами и обслуженными ими заказами. Используется для построения аналитики по доставке и для соединения сателлитов курьера с заказом.

#### Источник
- `ref('h_courier')` — связан логически: линк ссылается на хаб (зависимость задана в `depends_on`).
- `ref('h_order')` — связан логически: линк ссылается на хаб (зависимость задана в `depends_on`).
- `source('paas_content_demand__mindbox', 'spree_orders')` — фактический источник пар `(courier_id, order_id)`.

#### Схема
Модель материализуется в схеме `cdm`.

#### Материализация
`incremental`. При обычном запуске в таблицу добавляются только новые комбинации ключей: входящий поток `LEFT JOIN`-ится с целевой таблицей по `link_id`, после чего вставляются только пары, которых ещё нет. Полная перезагрузка возможна через `--full-refresh`.

#### Особенности
- `link_id` — суррогатный ключ связи: `md5(upper(trim(courier_id)) || '||' || upper(trim(order_id)))`.
- `courier_id` и `order_id` — суррогатные ключи соответствующих хабов.
- `mt_src_id` — фиксированное значение `MINDBOX`.
- Тесты: `unique` и `not_null` на `link_id` (см. `models/schemas/cdm/l_courier_x_order.yml`).
