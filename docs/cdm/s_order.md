#### Описание модели
Сателлит хаба "Заказ". Хранит описательные атрибуты заказа (стоимость, дата создания, дата отгрузки, тип доставки) и историю их изменений во времени. Используется в витринах, отвечающих за анализ заказов и отгрузок.

#### Источник
- `ref('h_order')` — сателлит расширяет хаб (зависимость задана в `depends_on`).
- `source('paas_content_demand__mindbox', 'spree_orders')` — источник атрибутов заказа.

#### Схема
Модель материализуется в схеме `cdm`.

#### Материализация
Кастомная — `scd_materialization` (см. `macros/materializations/scd_materialization.sql`). Логика загрузки:
- При первом запуске или `--full-refresh` целевая таблица создаётся через CTAS из временной таблицы.
- При инкрементальном запуске:
  - вставляются записи с новыми бизнес-ключами;
  - вставляются записи, у которых для существующего ключа изменился `mt_hash_diff`;
  - у предыдущей активной версии проставляется `mt_valid_to_dttm = новый mt_valid_from_dttm` и `mt_active_flg = false`;
  - записи, которых уже нет в источнике, помечаются как удалённые.

Параметр материализации: `hub_hash_key: order_id`.

#### Особенности
- `mt_hash_diff` — `md5` от конкатенации `order_price`, `order_created_dttm`, `order_shipped_dttm`, `shipment_type`.
- `mt_valid_from_dttm` устанавливается равным `now()` (моменту загрузки).
- `mt_valid_to_dttm` инициализируется через макрос `mt_valid_to_dttm()` (значение «бесконечности»).
- `mt_src_id` — фиксированное значение `MINDBOX`.
- Тесты: проверка уникальности пары `(order_id, mt_valid_from_dttm)`, валидность диапазона `mt_valid_from_dttm <= mt_valid_to_dttm` (см. `models/schemas/cdm/s_order.yml`).
