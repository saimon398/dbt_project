#### Описание модели
Таблица-хаб "Курьер". Хранит уникальный набор бизнес-ключей сущности "Курьер" и их суррогатные хеш-ключи. Используется как точка входа в Data Vault для линков и сателлитов, относящихся к курьерам службы доставки.

#### Источник
Источником служит таблица `source('paas_content_demand__shopper', 'spree_couriers')` (поле `courier_id` — бизнес-ключ).

#### Схема
Модель материализуется в схеме `cdm`.

#### Материализация
`incremental`. При обычном запуске в таблицу добавляются только новые бизнес-ключи: входящий поток `LEFT JOIN`-ится с целевой таблицей по `courier_id`, после чего вставляются только записи, для которых соответствия не нашлось. Полная перезагрузка возможна через `--full-refresh`.

#### Особенности
- `courier_id` — суррогатный ключ, рассчитанный как `md5(upper(trim(courier_id)))`.
- `courier_src_id` — оригинальный бизнес-ключ из источника.
- `mt_src_id` — фиксированное значение `SHOPPER`.
- `mt_load_dttm` — момент загрузки записи.
- На колонку `courier_id` навешены тесты `unique` и `not_null` (см. `models/schemas/cdm/h_courier.yml`).
