#### Описание модели
Таблица-хаб "Ритейлер". Хранит уникальный набор бизнес-ключей сущности "Ритейлер" и их суррогатные хеш-ключи. Используется как точка входа в Data Vault для линков и сателлитов, связанных с ритейлерами (партнёрами/магазинами), участвующими в обработке заказов.

#### Источник
Источником служит таблица `source('paas_content_demand__mindbox', 'spree_orders')` (поле `retailer_code` — бизнес-ключ ритейлера, фигурирующий в строках заказов).

#### Схема
Модель материализуется в схеме `cdm`.

#### Материализация
`incremental`. При обычном запуске в таблицу добавляются только новые бизнес-ключи: входящий поток `LEFT JOIN`-ится с целевой таблицей по `retailer_id`, после чего вставляются только записи, для которых соответствия не нашлось. Полная перезагрузка возможна через `--full-refresh`.

#### Особенности
- `retailer_id` — суррогатный ключ, рассчитанный как `md5(upper(trim(retailer_code)))`.
- `retailer_src_id` — оригинальный бизнес-ключ из источника (`retailer_code`).
- `mt_src_id` — фиксированное значение `MINDBOX`.
- `mt_load_dttm` — момент загрузки записи.
- На колонку `retailer_id` навешены тесты `unique` и `not_null` (см. `models/schemas/cdm/h_retailer.yml`).
