#### Описание модели
Таблица-хаб "Заказ". Хранит уникальный набор бизнес-ключей сущности "Заказ" и их суррогатные хеш-ключи. Используется как точка входа в Data Vault для всех линков и сателлитов, описывающих заказ (связи с пользователем, ритейлером, курьером и т. д.).

#### Источник
Источником служит таблица `source('paas_content_demand__mindbox', 'spree_orders')` (поле `order_id` — бизнес-ключ).

#### Схема
Модель материализуется в схеме `cdm`.

#### Материализация
`incremental`. При обычном запуске в таблицу добавляются только новые бизнес-ключи: входящий поток `LEFT JOIN`-ится с целевой таблицей по `order_id`, после чего вставляются только записи, для которых соответствия не нашлось. Полная перезагрузка возможна через `--full-refresh`.

#### Особенности
- `order_id` — суррогатный ключ, рассчитанный как `md5(upper(trim(order_id)))`.
- `order_src_id` — оригинальный бизнес-ключ из источника.
- `mt_src_id` — фиксированное значение `MINDBOX`.
- `mt_load_dttm` — момент загрузки записи.
- На колонку `order_id` навешены тесты `unique` и `not_null` (см. `models/schemas/cdm/h_order.yml`).
