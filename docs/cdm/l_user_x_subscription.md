#### Описание модели
Таблица-линк "Пользователь — Подписка". Хранит уникальные связи между пользователями и их подписками в программе лояльности. Используется для аналитики по подпискам и для соединения сателлитов подписки с пользователем.

#### Источник
- `ref('h_user')` — связан логически: линк ссылается на хаб (зависимость задана в `depends_on`).
- `ref('h_subscription')` — связан логически: линк ссылается на хаб (зависимость задана в `depends_on`).
- `source('paas_content_demand__loyalty', 'loyalty_subscriptions')` — фактический источник пар `(crm_user_id, crm_subscription_id)`.

#### Схема
Модель материализуется в схеме `cdm`.

#### Материализация
`incremental`. При обычном запуске в таблицу добавляются только новые комбинации ключей: входящий поток `LEFT JOIN`-ится с целевой таблицей по `link_id`, после чего вставляются только пары, которых ещё нет. Полная перезагрузка возможна через `--full-refresh`.

#### Особенности
- `link_id` — суррогатный ключ связи: `md5(upper(trim(crm_subscription_id)) || '||' || upper(trim(crm_user_id)))`.
- `subscription_id` и `user_id` — суррогатные ключи соответствующих хабов.
- `mt_src_id` — фиксированное значение `LOYALTY`.
- Тесты: `unique` и `not_null` на `link_id` (см. `models/schemas/cdm/l_user_x_subscription.yml`).
