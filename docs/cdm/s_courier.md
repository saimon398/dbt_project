#### Описание модели
Сателлит хаба "Курьер". Хранит описательные атрибуты курьера (имя, фамилия, тип) и историю их изменений во времени. Используется для получения актуального либо исторического состояния курьера на заданную дату.

#### Источник
- `ref('h_courier')` — сателлит расширяет хаб (зависимость задана в `depends_on`).
- `source('paas_content_demand__shopper', 'spree_couriers')` — источник атрибутов курьера.

#### Схема
Модель материализуется в схеме `cdm`.

#### Материализация
Кастомная — `scd_materialization` (см. `macros/materializations/scd_materialization.sql`). Логика загрузки:
- При первом запуске или `--full-refresh` целевая таблица создаётся через CTAS из временной таблицы.
- При инкрементальном запуске:
  - вставляются записи с новыми бизнес-ключами;
  - вставляются записи, у которых для существующего ключа изменился `mt_hash_diff`;
  - у предыдущей активной версии проставляется `mt_valid_to_dttm = новый mt_valid_from_dttm` и `mt_active_flg = false`;
  - записи, которых уже нет в источнике, помечаются как удалённые.

Параметр материализации: `hub_hash_key: courier_id`.

#### Особенности
- `mt_hash_diff` — `md5` от конкатенации `first_name`, `last_name`, `courier_type`.
- `mt_valid_from_dttm` устанавливается равным `now()` (моменту загрузки).
- `mt_valid_to_dttm` инициализируется через макрос `mt_valid_to_dttm()` (значение «бесконечности»).
- `courier_type` ограничен типом `varchar(4)`.
- `mt_src_id` — фиксированное значение `SHOPPER`.
- Тесты: проверка уникальности пары `(courier_id, mt_valid_from_dttm)`, валидность диапазона `mt_valid_from_dttm <= mt_valid_to_dttm` (см. `models/schemas/cdm/s_courier.yml`).
