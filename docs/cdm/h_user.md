#### Описание модели
Таблица-хаб "Пользователь". Хранит уникальный набор бизнес-ключей сущности "Пользователь" и их суррогатные хеш-ключи. Используется как точка входа в Data Vault для всех связанных линков и сателлитов, описывающих пользователя.

#### Источник
Источником служит таблица `source('paas_content_demand__mindbox', 'spree_users')` (поле `crm_user_id` — бизнес-ключ).

#### Схема
Модель материализуется в схеме `cdm`.

#### Материализация
`incremental`. При обычном запуске в таблицу добавляются только новые бизнес-ключи: входящий поток `LEFT JOIN`-ится с целевой таблицей по `user_id`, после чего вставляются только записи, для которых соответствия не нашлось. Полная перезагрузка возможна через `--full-refresh`.

#### Особенности
- `user_id` — суррогатный ключ, рассчитанный как `md5(upper(trim(crm_user_id)))`.
- `user_src_id` — оригинальный бизнес-ключ из источника.
- `mt_src_id` — фиксированное значение `MINDBOX`.
- `mt_load_dttm` — момент загрузки записи.
- На колонку `user_id` навешены тесты `unique` и `not_null` (см. `models/schemas/cdm/h_user.yml`).
