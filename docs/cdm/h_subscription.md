#### Описание модели
Таблица-хаб "Подписка". Хранит уникальный набор бизнес-ключей сущности "Подписка" и их суррогатные хеш-ключи. Используется как точка входа в Data Vault для линков и сателлитов, описывающих подписки пользователей в программе лояльности.

#### Источник
Источником служит таблица `source('paas_content_demand__loyalty', 'loyalty_subscriptions')` (поле `crm_subscription_id` — бизнес-ключ).

#### Схема
Модель материализуется в схеме `cdm`.

#### Материализация
`incremental`. При обычном запуске в таблицу добавляются только новые бизнес-ключи: входящий поток `LEFT JOIN`-ится с целевой таблицей по `subscription_id`, после чего вставляются только записи, для которых соответствия не нашлось. Полная перезагрузка возможна через `--full-refresh`.

#### Особенности
- `subscription_id` — суррогатный ключ, рассчитанный как `md5(upper(trim(crm_subscription_id)))`.
- `subscription_src_id` — оригинальный бизнес-ключ из источника.
- `mt_src_id` — фиксированное значение `LOYALTY`.
- `mt_load_dttm` — момент загрузки записи.
- На колонку `subscription_id` навешены тесты `unique` и `not_null` (см. `models/schemas/cdm/h_subscription.yml`).
